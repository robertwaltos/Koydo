# Evidence Archive Vendor Cost Comparison (Launch Baseline)

Date prepared: 2026-03-03

## Estimated needs used for this comparison
- Compliance/admin users needing legal hold and audit export access: **10 seats**
- New evidence volume generated by automation/reporting: **~5 GB/month**
- Retention policy target: **7 years (2555 days)**
- Background compliance automation cadence: daily scheduler with randomized execution, hard max 5-day gap

## Vendor comparison (Google-first decision context)
| Vendor option | Public pricing signal | What it covers | Planning cost for baseline needs |
| --- | --- | --- | --- |
| Google Vault (Google Workspace) | Vault availability is tied to eligible Workspace editions/add-ons; Business Plus is listed at **$14 USD/user/month** on annual commitment on the pricing page. | Native legal hold/eDiscovery for Google Workspace data and export workflows. | **~$140/month** for 10 seats (Workspace Business Plus only). |
| Microsoft Purview eDiscovery route | Purview licensing requires eligible Microsoft 365/E5/A5/G5 foundations; Purview Suite page advertises **$12 USD/user/month** for the suite. | eDiscovery, audit, data governance stack in Microsoft ecosystem. | **~$120/month incremental** for 10 seats if Suite add-on is required, plus base M365 licensing. |
| Box Governance Legal Holds | Box docs state Governance can be added to supported enterprise plans; public governance add-on price is typically quote-based. | Legal hold and governance controls for content already stored in Box. | **Quote required** (plan as base Box seats + Governance add-on). |
| Relativity Legal Hold | Relativity pricing is sales-led (public “contact us” flow). | Enterprise legal-hold + legal-ops workflow platform. | **Quote required** (usually enterprise procurement path). |

## Immutable/WORM storage pairing costs
Pairing legal-hold tooling with immutable object storage remains the recommended model for cross-platform evidence archives.

| Immutable option | Pricing signal | Practical implication for this app |
| --- | --- | --- |
| AWS S3 Object Lock | S3 Object Lock uses standard S3 storage/request economics; request charges apply (AWS examples show per-1K PUT/GET style pricing), and lifecycle/batch operations add cost. | Good fit for low-to-medium artifact volumes; costs scale mainly with retained GB-month and request volume. |
| AWS Backup Vault Lock | AWS docs state **no additional charge** for Vault Lock; standard AWS Backup storage/usage charges still apply. | Strong for backup-centric compliance retention with immutability controls. |
| Azure Blob immutable storage | Microsoft docs indicate **no extra capacity charge** for enabling immutable storage; write-operation transaction charges still apply. | Cost-efficient for long retention if using Azure storage as evidence sink. |

## Recommended launch choice
- **Primary choice**: Google-first path (`google_vault_export`) because the current operating stack already relies on Google Workspace.
- **Architecture**: keep evidence provider abstraction pluggable (already implemented) so migration to Purview/Box/Relativity remains possible without redesign.

## Procurement notes
- Box Governance and Relativity Legal Hold should be treated as RFQ items; no reliable public per-seat add-on list pricing was available at time of review.
- Purview total cost depends on existing Microsoft licensing baseline (already-E5 vs add-on path).
- Storage retention costs should be modeled separately from legal-hold seat licensing.
